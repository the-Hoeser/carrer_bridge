## Overview

The Attendance Management System is a full-stack, monorepo web application designed for college environments. It supports a multi-level institutional hierarchy (Department -> Year -> Division -> Batch) and complex Role-Based Access Control (RBAC).

```
│        [Data Layer] Prisma + Postgres   │
└─────────────────────────────────────────┘
```

## Components

### Backend API (`backend/`)
- **Purpose:** Core business logic, authentication, and data management.
- **Location:** `backend/src`
- **Patterns:** Controller-Service-Repository (via Prisma).
- **Key Modules:**
  - `auth`: JWT-based auth with OTP first-login flow.
  - `structure`: Management of departments, years, divisions, and batches.
  - `attendance`: Slot-based attendance marking and locking logic.
  - `timetable`: Versioned timetable management with conflict validation.

### Frontend Application (`frontend/`)
- **Purpose:** User interface for Students, Faculty, and Admins.
- **Location:** `frontend/src`
- **Key Sections:**
  - `context/AuthContext.tsx`: Manages session state and multi-step login.
  - `services/api.ts`: Centralized Axios instance for backend communication.
  - `pages/`: Dashboard implementations for various roles.

## Data Flow

1. **Authentication:** User logs in via API -> Receives JWT -> Stored in LocalStorage -> Interceptor adds to all requests.
2. **Attendance Marking:** Teacher fetches today's schedule -> Marks students -> API validates lock status -> Prisma transaction updates DB.
3. **Analytics:** Dashboards fetch aggregated data via `/api/v1/analytics` with role-scoped filters.
